# Example: Project 1 Marketing Operations

This example describes how Project 2 can later support Project 1-style
automation review and documentation workflows.

Project 1 is a completed AI Marketing Operations Agent. It includes browser
automation, REST and GraphQL integrations, deterministic aggregation, anomaly
detection, reporting, run history, human approval and mock notification
delivery.

Project 2 should not duplicate Project 1's business workflow. Instead, it can
provide reusable tooling around Project 1-style needs:

- review workflow boundaries and safety controls;
- generate or improve runbooks;
- investigate CI failures;
- design deterministic MCP tools for repeatable local operations;
- share Codex and Claude Code guidance for agent-assisted engineering.

This example does not call Project 1 code and does not add integrations.
